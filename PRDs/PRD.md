产品需求文档 (PRD)：快找房 (Web端)
1. 项目概述

产品名称：快找房 


产品目标：通过清晰的房屋信息展示，让租房者快速挑选到合适房源 。


核心业务模块：登录页模块、首页模块、房源模块、用户模块 。

技术栈建议：前端建议采用 React (或 Vue 3) + TypeScript + Tailwind CSS；后端服务采用 C# .NET Core 构建 RESTful API，充分保证系统的性能与可维护性。

2. 全局规范与UI布局
系统框架设计：采用经典的后台管理系统“左右-上下”经典布局。


左侧导航栏：固定显示，包含菜单项“房源模块”和“用户模块”，点击切换右侧视图 。


右侧顶部 Header：全局展示区，实时展示天气信息、当前时间，以及当前登录用户信息（例如：“欢迎你，王凯明”） 。

右侧主体内容区：根据左侧路由动态渲染页面内容。

通用操作区布局：在房源和用户列表页，顶部操作栏统一遵循 “左侧为搜索查询栏，右侧为新增按钮” 的排版原则。

3. 详细功能模块说明
3.1 登录模块 (Login Module)
功能描述：系统的入口，用于验证管理员或业务员的身份。

页面元素：包含系统名称“快找房”Logo/标题，账号输入框、密码输入框、登录按钮。


交互逻辑：登录成功后，全局 Header 获取并展示当前登录用户信息，页面自动跳转至“首页模块” 。

3.2 首页模块 (Home Module)
功能描述：用户成功登录后的默认工作台。


核心内容：右侧主体区域默认直接展示“房源模块”的列表信息，以便用户第一时间开展核心业务 。

3.3 房源模块 (Housing Module)

功能描述：用于管理系统内的房屋租赁详细信息 。

顶部操作栏：


左侧搜索查询：提供组合查询表单，支持根据 小区名称 / 上架时间 / 价格 进行条件筛选 。


右侧主要操作：提供“添加房源”按钮，点击唤出统一动态表单弹窗 。


数据展示 (Data Table)：以表格形式呈现，包含表头字段：小区名称、房龄、上架时间、价格 。


行级操作：表格每一行末尾固定提供“编辑”与“删除”按钮，用于维护单条房源数据 。

3.4 用户模块 (User Module)

功能描述：用于管理系统内的客户或租客信息 。

顶部操作栏：


左侧搜索查询：提供组合查询表单，支持根据 用户名称 / 添加时间 / 联系方式 进行条件筛选 。


右侧主要操作：提供“添加用户信息”按钮，点击唤出统一动态表单弹窗 。


数据展示 (Data Table)：以表格形式呈现，包含表头字段：用户名称、用户性别、添加时间、联系方式 。


行级操作：表格每一行末尾固定提供“编辑”与“删除”按钮，用于维护单条用户数据 。

3.5 统一动态表单组件 (Dynamic Form)

功能描述：添加用户和添加房屋统一复用该弹窗组件，通过传入类型属性动态渲染内容 。

动态渲染逻辑：根据传入的 type（"house" 或 "user"），智能切换标题、字段 Label 及输入控件类型：
| 表单行 | type="house" (房源模式) | type="user" (用户模式) | 建议控件类型 |
| :--- | :--- | :--- | :--- |
| 字段 1 | 小区名称  | 姓名  | 纯文本输入框 (Input) |
| 字段 2 | 小区价格  | 性别  | 房源为数字输入(Number)；用户为下拉选择(Select) |
| 字段 3 | 房龄  | 联系方式  | 房源为数字输入(Number)；用户为文本输入(Input) |


底部操作：包含“确定”与“取消”按钮，提交时需进行基础的必填项非空校验 。
