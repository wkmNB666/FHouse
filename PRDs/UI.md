UI 设计规范文档 (UI Design Specification)
1. 整体视觉风格与色彩规范 (Visual Style)
本项目为效率导向的后台管理系统，建议采用现代、简洁、扁平化的设计风格（类似 Ant Design 或 Element Plus 的默认风格）。

主色调 (Primary Color)：建议使用科技蓝（如 #1890ff），传达专业、可靠的“找房/业务管理”属性。用于主按钮（添加）、选中状态和强调链接。

辅助色 (Functional Colors)：

成功：绿色（#52c41a），用于操作成功提示。

危险/警告：红色（#ff4d4f），用于“删除”按钮或错误提示。

中性色 (Neutral Colors)：

背景色：浅灰色（#f0f2f5），用于右侧主体内容的底层背景，与白色的内容卡片形成对比。

文字颜色：主要文字使用深灰（#333333），次要文字/占位符使用浅灰（#999999）。

2. 全局布局规范 (Layout Specification)
采用经典的 Sider + Header + Content 结构：

左侧导航 (Sider)：固定宽度（建议 200px - 240px），深色背景（#001529）或纯白背景带右侧阴影。选中当前菜单时，背景高亮并带有一条左侧或右侧的蓝色指示线。

顶部标头 (Header)：高度固定（建议 64px），纯白背景，底部带浅色分割线。左右两端对齐，右侧区域各元素（天气、时间、用户信息）之间保持适当的间距（如 16px）。

主体内容区 (Content)：外围留白（建议 24px 的 Padding），内部为纯白色的卡片（Card）容器，带微小圆角（4px 或 8px），所有业务表格和操作区均在这个白色卡片内展示。

3. 组件交互规范 (Component Interaction)
顶部操作区：

搜索框统一带“放大镜”图标。

“查询”按钮与搜索框紧贴或保持固定间距。

“添加”类主要按钮放置在最右侧，使用主色调实心按钮。

数据表格 (Table)：

表头（Th）背景加浅灰，字体加粗，区分数据行。

数据行鼠标悬浮（Hover）时，整行背景变浅灰，引导视觉焦点。

“编辑”使用蓝色文字按钮，“删除”使用红色文字按钮，避免视觉过度拥挤。

动态弹窗 (Modal Form)：

点击“添加”或“编辑”时，从屏幕中间弹出，底层带半透明黑色遮罩（Mask）。

表单输入框失去焦点或点击“确定”时，触发非空校验提示（标红并提示“此项为必填”）。
